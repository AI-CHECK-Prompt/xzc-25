# 启动指南

## 1. 环境要求

- Docker 20.10+
- Docker Compose v2+

## 2. 一键启动

```bash
docker compose up -d --build
```

启动顺序：

1. `postgres` 等待健康检查通过
2. `redis` 等待健康检查通过
3. `backend` 建表 + 灌入种子数据（六方参与方、6 个示例账号、2 个示例项目）
4. `frontend` 构建并由 nginx 提供
5. `simulator` 在 backend 就绪后批量注入数据

## 3. 验证

访问：

- 前端门户 <http://localhost:5173>
- 后端 API 文档 <http://localhost:8000/docs>
- 健康检查 `curl http://localhost:8000/`

自检：

```bash
docker compose exec backend python -m app.self_check
```

期望输出 4/4 通过。

## 5. 停止与清理

```bash
docker compose down            # 停止容器
docker compose down -v         # 停止并删除数据卷（彻底清理）
```

## 6. 故障排查

| 现象 | 处理 |
|---|---|
| 8000 端口占用 | 修改 `docker-compose.yml` 中 backend 端口映射 |
| 前端 502 | backend 尚未就绪，等待 30 秒 |
| 模拟器连接失败 | 确认 backend 容器名与 API_BASE 一致 |
