# 数据模型

## 关键实体

| 表 | 中文 | 关键字段 |
|---|---|---|
| parties | 参与方 | code, name, role（factory/transport/contractor/supervisor/owner/quality） |
| users | 用户 | username, password_hash, role, party_id |
| projects | 项目 | code, name, owner_party_id, contractor_party_id, supervisor_party_id |
| components | 构件 | trace_code（唯一追溯码）, rfid_tag, mould_no, rebar_batch, concrete_ratio, pour_at, factory_inspection, current_stage |
| factory_out_records | 出厂记录 | out_at, vehicle_no, driver, route_plan, inspection_conclusion |
| transport_telemetry | 运输遥测 | reported_at, longitude, latitude, temperature, humidity, status |
| transport_alerts | 运输告警 | alert_type（OFF_ROUTE / OVERSTAY / TEMP_OUT） |
| site_entry_records | 进场记录 | entered_at, stack_location, inspector, acceptance |
| hoisting_records | 吊装记录 | equipment_no, signal_worker, rigger, coord_lng, coord_lat, result |
| joint_connections | 节点连接 | grout_at, grout_batch, connection_type, operator |
| concealed_acceptances | 隐蔽验收 | quality_grade, inspector, conclusion |
| protection_records | 成品保护 | decoration, mep, measures, risk_warning |
| archive_packages | 档案包 | archive_no, file_path, status, submitted_at, accepted_at |
| offline_sync_logs | 离线同步日志 | client_id, batch_id, payload, status |

## 唯一追溯码生成规则

`PC-YYYYMMDD-{工厂代码}-{4 位序列号}`

- 工厂代码取自 parties.code 大写
- 当日按工厂递增
- 同时生成 RFID 标签 `RFID-{追溯码}` 与二维码内容 `http://.../trace?code=...`

## 关键流程

```
已生产 → 出厂 → 运输中 → 已到场 → 已进场 → 已吊装 → 节点连接 → 已隐蔽 → 成品保护 → 档案归档
                                                                                  ↓
                                                                          进场不合格(旁路)
```
