## 唯一追溯码生成规则

`PC-YYYYMMDD-{工厂代码}-{4 位序列号}`

- 工厂代码取自 parties.code 大写
- 当日按工厂递增
- 同时生成 RFID 标签 `RFID-{追溯码}` 与二维码内容 `http://.../trace?code=...`

## 关键流程

```
已生产 → 出厂 → 运输中 → 已到场 → 已进场 → 已吊装 → 节点连接 → 已隐蔽 → 成品保护 → 档案归档
                  ↓                                                                    ↓
              运输告警                                                              运营期维护
                  ↓
            质监发起抽检 → 现场录入 → 不合格则自动开整改 → 阻断下游工序 → 整改完成 → 申请复核 → 原抽检人复核 → 解锁
```

## 维护周期建议规则

`建议周期 = 基础周期（按构件类型）× 位置系数 × 最近发现系数`

- 基础周期：外墙板 180d / 整体卫浴 90d / 其他 365d
- 位置系数：屋面/卫生间 0.6、外墙/厨房 0.7
- 发现系数：正常 1.0、轻微异常 0.6、严重异常 0.3（最低 30 天）
- 风险等级：正常 → low，严重异常 → high

## 抽检阻断规则

`has_blocking_inspection(component_id, target_stage)`：

1. 找出该构件所有未闭环的抽检任务
2. 若任务的抽检工序 rank < 目标工序 rank，且最新记录结论 = 不合格
3. 且对应整改单状态 ≠ 已闭环
4. → 返回 true，阻止该目标工序的写入
